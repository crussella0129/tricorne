# Code of Conduct

Tricorne adopts the **Fedora Project Code of Conduct** verbatim.

## Canonical source

The authoritative text is maintained by the Fedora Council and published at:

<https://docs.fedoraproject.org/en-US/project/code-of-conduct/>

Where this file and the Fedora canonical text differ, **the Fedora
canonical text governs**. If you notice a discrepancy, open an issue
and we will update.

> **Pre-v0.1 note.** Before public launch, the full Fedora CoC text will
> be mirrored into this file verbatim. The URL above is the definitive
> source until that mirroring happens.

## Why Fedora's CoC

Tricorne is a Fedora Remix. Our contributor community substantially
overlaps with Fedora's, and we target becoming an official Fedora Spin.
Using the same Code of Conduct — verbatim, with the same enforcement
principles — removes ambiguity for contributors who work across both
communities and signals upstream-first intent.

## Scope

This Code of Conduct applies to all Tricorne project spaces, including:

- This repository (issues, pull requests, discussions, commit messages,
  code comments)
- Any official Tricorne communication channel (mailing list, chat,
  forum) established in the future
- Any public event where someone represents Tricorne (conference talks,
  booth staffing, workshops)

It also applies when a member's behavior in adjacent spaces materially
affects the safety or productivity of the Tricorne community.

## Reporting

Until Tricorne establishes a dedicated CoC committee (target: v0.3,
when the project transitions from BDFL to working-group governance),
Code of Conduct concerns should be reported to:

- **Charles Russella** (project BDFL): `crussella0129@gmail.com`

If the report involves Charles, or if you would prefer an external
channel, you may escalate to the **Fedora Council's Code of Conduct
process** at
<https://docs.fedoraproject.org/en-US/project/code-of-conduct/#_reporting>.

All reports are treated as confidential. The full triage, response, and
consequences process follows Fedora's — see the canonical URL.

## The short version

Be considerate. Be respectful. Be collaborative. When you disagree,
disagree on the substance without attacking the person. Assume good
faith and, when you cannot, ask a question rather than making an
accusation. Step back when you need to.

Tricorne is an offensive security project. Our subject matter is
adversarial, but our community is not. If the distinction between a
simulated attack and a personal attack is unclear to you, err on the
side of kindness.
